# Telegram Group Bot

Личный Telegram-бот для использования в группе с друзьями.  
Написан на Python (`pyTelegramBotAPI`), задеплоен на **Render** (бесплатный тариф), база данных — **Neon** (бесплатный PostgreSQL).

---

## Стек

| Компонент | Технология |
|-----------|------------|
| Язык | Python 3.12+ |
| Telegram-библиотека | pyTelegramBotAPI (telebot) |
| База данных | PostgreSQL (Neon) |
| Хостинг | Render (Free Web Service) |
| Мониторинг | UptimeRobot (пингует каждые 5 минут) |
| Веб-сервер | Flask (мини-сервер, только чтобы Render не усыплял сервис) |

---

## Структура проекта

```
├── main.py        — пользовательские хендлеры, middleware статистики, Flask-сервер, точка входа
├── admin.py       — все команды для администраторов (стата, топ, юзер и др.)
├── config.py      — объект bot, переменные окружения, логирование
├── database.py    — вся работа с PostgreSQL (подключение, таблицы, запросы)
├── utils.py       — хелперы (парсер времени, форматирование, упоминания, аптайм, отчёты)
└── requirements.txt
```

Новая пользовательская фича = новые функции в `database.py` (если нужна БД) + новые хендлеры в `main.py`.  
Новая админ-фича = функции в `database.py` + хендлеры в `admin.py`.  
`config.py` и `utils.py` трогаются редко.

## Переменные окружения

Задаются в **Render → Environment**:

| Переменная | Обязательна | Описание |
|------------|-------------|----------|
| `BOT_TOKEN` | ✅ | Токен бота от @BotFather |
| `DATABASE_URL` | ✅ | Строка подключения к Neon PostgreSQL (с `-pooler` в хосте) |
| `ADMIN_IDS` | ✅ | Telegram ID администраторов через запятую, например `123456789,987654321` |

Узнать свой Telegram ID — написать боту `/myid`.

---

## База данных

Таблицы создаются автоматически при первом запуске (`database.init_db()`).

| Таблица | Что хранит |
|---------|------------|
| `timers` | Активные таймеры (переживают перезапуск бота) |
| `users` | Все пользователи, которых бот видел в чатах |
| `chats` | Все чаты, где бот присутствует |
| `user_chat_stats` | Счётчики активности каждого пользователя в каждом чате |

## Команды бота

| Команда | Синонимы | Описание |
|---------|----------|----------|
| `/start`, `/help` | — | Справка |
| `/ping` | — | Ping (round-trip до Telegram API) + Uptime |
| `/myid` | — | Показать свой Telegram ID |
| `/t [время] [описание]` | `/timer`, `таймер`, `тайм` | Установить таймер |
| `/mytimers` | `таймеры` | Список своих активных таймеров |
| `/del [ID]` | `/cancel`, `удалить`, `отмена` | Удалить таймер по ID |

### Админ-команды (только в личке, только для ADMIN_IDS)

| Команда | Описание |
|---------|----------|
| `адмхелп` | Список всех админ-команд |
| `стата` / `статистика` | Общая статистика бота |
| `топ вся` | Глобальный топ по всем чатам (с пагинацией ◀️▶️) |
| `топ [название или ID чата]` | Топ пользователей в конкретном чате |
| `юзер [ID или @username]` | Детальная статистика конкретного пользователя |

### Формат времени для таймера

Поддерживаются русские и английские единицы:

```
д / d — дни
ч / h — часы
м / m — минуты
с / s — секунды
```

Можно комбинировать: `1д5ч30м`, `2h30m`, `10с`, `45s`.

### Команда стата

Показывает:
- Общее количество пользователей, чатов, сообщений, символов, стикеров, фото, видео, голосовых, GIF
- Топ-10 пользователей по активности с разбивкой по чатам

Доступна **только в личке** бота и **только для ID из `ADMIN_IDS`**.

## Важные технические детали

**Таймеры**  
Реализованы через `threading.Timer` — не блокируют polling.  
Сохраняются в БД при создании, удаляются при срабатывании/отмене.  
При перезапуске бота восстанавливаются из БД. Если таймер должен был сработать пока бот был выключен — срабатывает сразу с пометкой об опоздании.

**Статистика**  
Middleware перехватывает каждое входящее сообщение и записывает счётчики в БД **в фоновом потоке** (чтобы не задерживать ответ бота).  
Для работы статистики в группах необходимо отключить Privacy Mode бота:  
`@BotFather → /mybots → Bot Settings → Group Privacy → Turn off`

**Отказоустойчивость**  
Polling запущен в `while True` с `try/except` — при любой сетевой ошибке (включая периодические `Connection reset by peer` со стороны Telegram) бот автоматически переподключается через 5 секунд.

**Бесплатный хостинг — нюансы**  
- Render: 750 бесплатных часов/месяц на воркспейс. Одного бота хватает на весь месяц.  
- Neon: база "засыпает" после 5 минут без запросов, первый запрос после паузы +300-500мс (cold start). На бесплатном тарифе это не отключается.  
- UptimeRobot не даёт Render-сервису засыпать, но от перезапусков самого Render не защищает.

## Деплой (с нуля)

1. Создать бота через @BotFather, получить токен.
2. Отключить Privacy Mode: `@BotFather → /mybots → Bot Settings → Group Privacy → Turn off`.
3. Создать проект на [neon.tech](https://neon.tech), скопировать **pooled** connection string.
4. Загрузить все файлы в GitHub-репозиторий (можно приватный).
5. На [render.com](https://render.com) → New → Web Service → подключить репозиторий:
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python main.py`
   - **Instance Type**: Free
6. Добавить переменные окружения: `BOT_TOKEN`, `DATABASE_URL`, `ADMIN_IDS`.
7. Задеплоить. Написать боту `/myid`, убедиться что возвращает ID.
8. Настроить [UptimeRobot](https://uptimerobot.com): HTTP monitor на URL сервиса с Render, интервал 5 минут.
